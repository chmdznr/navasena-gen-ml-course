RAG Fundamentals

RAG membekali LLM dengan konteks faktual dari dokumenmu (embed → retrieve → rerank → generate) sebelum menjawab — halusinasi berkurang, jawaban bisa di-update tanpa latih ulang, dan setiap klaim bisa disitasi. Ringkasan delapan notebook konsep.
Alur: Ingest PDF (Docling) → Chunk + halaman/heading → Embed (MiniLM multilingual) → FAISS retrieve (top-20) → Rerank (bge-reranker → top-4) → Augment: konteks + pertanyaan → Generate (NIM/Qwen) → Jawaban + sitasi [S#]

① Fundamentals

  • LLM polos: halusinasi + knowledge cutoff
  • RAG = Retrieve (cari konteks) + Generate (LLM jawab)
  • Embed multilingual → cari makna, bukan kata
  • Tambah pengetahuan = tambah dokumen (tanpa latih ulang)
▸ embed → retrieve → generate

② Ingest & Chunk

  • Docling: PDF → struktur (heading, tabel) + OCR
  • HybridChunker: potong sesuai struktur, tiap chunk ≤ batas token
  • Simpan halaman + heading tiap chunk → bahan sitasi
  • Embed versi contextualize (heading + teks), bukan teks polos
▸ ingest baik = retrieval baik

③ Retrieve Better (Rerank)

  • Bi-encoder + FAISS: cepat, ambil 20 kandidat
  • Cross-encoder rerank: akurat, urutkan ulang → top-4
  • Model: bge-reranker-v2-m3 (multilingual)
  • Reranker menilai relevansi topik, bukan menjamin jawaban ada di konteks
▸ presisi konteks naik

④ Ukur (RAGAS)

  • Retrieval: Context Precision, Context Recall
  • Jawaban: Faithfulness (anti-ngarang), Answer Relevancy
  • Judge harus kuat → NVIDIA NIM (Llama-3.3-70B)
  • Model kecil sering gagal parsing JSON → skor NaN
▸ tahu di mana yang lemah

⑤ Skala Index

  • Flat (exact, recall 100%) · IVF/HNSW (perkiraan, cepat)
  • Cosine: normalize_L2 dulu lalu IndexFlatIP
  • Persist: FAISS write_index / ChromaDB PersistentClient
  • Trade-off: tukar sedikit recall untuk kecepatan
▸ saat korpus membesar

⑥ Conversational RAG

  • Pertanyaan lanjutan (elipsis/kata ganti) → retrieval gagal
  • History-aware rewriting: tulis ulang jadi mandiri dulu
  • Memori window (terbaru) + summary (lama diringkas)
  • Contoh: "Kalau cuti sakit?" → "Berapa hari cuti sakit?"
▸ percakapan multi-giliran

⑦ Capstone: Sitasi

  • Tiap konteks diberi label [S#] + halaman/heading
  • LLM bubuhkan [S#] pada klaim → "Lihat sumber"
  • Verifikasi: cited_labels tandai label di luar rentang
  • Ask-My-Document: PDF → tanya → jawaban terverifikasi
▸ jawaban bisa ditelusuri

⑧ Deploy (/ask)

  • FastAPI /ask + Pydantic (input cacat → HTTP 422)
  • RagEngine: muat model sekali saat start
  • Generator: NIM (cloud, tanpa GPU) / Qwen (lokal T4)
  • Smoke test in-process TestClient; runbook = deliverable
▸ RAG sebagai layanan

Istilah Kunci
RAG — Retrieval-Augmented Generation: bekali LLM konteks eksternal sebelum menjawab.
Embedding — Teks sebagai vektor; makna mirip → vektor berdekatan (multilingual = lintas-bahasa).
Chunk — Potongan dokumen (unit retrieval); HybridChunker memotong sadar-struktur.
Reranker — Cross-encoder (bge-reranker-v2-m3) yang menilai ulang relevansi kandidat dengan teliti.
RAGAS — Framework evaluasi RAG: Faithfulness, Answer Relevancy, Context Precision, Context Recall.
FAISS — Pencari vektor terdekat yang cepat; Flat (exact) vs IVF/HNSW (perkiraan).
Sitasi [S#] — Label sumber pada klaim; diverifikasi agar tak mengada-ada (cited_labels).
NIM — NVIDIA Inference Microservice; LLM cloud (Llama-3.3-70B) andal tanpa GPU lokal.
